## Install the package

```bash
composer require cerealkiller97/filament-bug-reports
```

## Publish and run the migration

```bash
php artisan vendor:publish --tag=bug-reports-migrations
php artisan migrate
```

Optionally publish the config file:

```bash
php artisan vendor:publish --tag=bug-reports-config
```

## Register the plugin

Add it to the panel you want it in:

```php title="app/Providers/Filament/AdminPanelProvider.php"
use CerealKiller97\FilamentBugReports\BugReportsPlugin;

public function panel(Panel $panel): Panel
{
    return $panel
        // ...
        ->plugin(
            BugReportsPlugin::make()
                ->authorizeManagementUsing(fn (User $user): bool => $user->isAdmin()),
        );
}
```

<Callout type="warning" title="Management defaults to nobody">
That `authorizeManagementUsing` call matters. Until you opt someone in, no one
can see the report list — though everyone can still file a report. See
[Authorization](/docs/authorization).
</Callout>

## Point it at a repository

Give it a token with the `repo` (or `issues:write`) scope, and the repository
issues should be created in:

```dotenv title=".env"
BUG_REPORTS_GITHUB_TOKEN=ghp_xxxxxxxxxxxx
BUG_REPORTS_GITHUB_REPOSITORY=acme/platform
```

Both fall back to `GITHUB_TOKEN` and `GITHUB_BUG_REPOSITORY` if you already have
those set. Everything else is covered in [Configuration](/docs/configuration).
